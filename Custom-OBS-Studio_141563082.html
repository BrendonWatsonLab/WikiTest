Watson Lab Behavioral Box : Custom OBS Studio

Created by Pho Hale, last modified on Jul 22, 2020

https://github.com/CommanderPho/obs-studio

A modified version of the open source "OBS Studio" software (https://obsproject.com/) that adds microsecond timestamp support. Used to record mouse videos from the Camera.

Installer Package (Windows 64-bit):

obs-studio-x64-24.0.3-371-g8961e488.zipobs-studio-x64-24.0.3-371-g8961e488.msi

Setup:

Navigate to File > Settings


Click on "Output" tab in the left sidebar

Click on "Recording" tab on the "Output" page

"File path or URL": Set to the path where you want the videos to be saved to by default.

You can also customize the other recording settings



Click on "Advanced" tab in the left sidebar


Under the "Recording" section on the "Advanced" page

"Filename Formatting": This sets the default filename for the recorded video files.Â  Change the "_B00_" to the correct BB ID for the connected camera.

You can also customize the other recording settings


Navigate to Tools > Scripts

Click on "autosplitter.lua" in the "Loaded Scripts" list on the left

If it does not appear in this list, download it according to the "Required Plugins" section and then click the "+" button on the scripts window to add it to the "Loaded Scripts" list. If it does appear, but selecting it does not populate the options in the right hand portion of the panel, click the "-" symbol to remove it, and then re-add it with "+".

You can adjust the "autosplitter" feature, which automatically stops and starts the recording to create a new file at a predefined interval. This will result in multiple files of uniform length instead of one long file.Â 

You can check the "Enabled" box to enable/disable this feature.

The "Seconds", "Minutes", "Hours" fields determine the maximum continuous recording time before autosplitting the file. These will only have an effect if the "Enabled" box is checked.

Required Dependencies

These are the things that must be installed before the custom OBS Studio software can be installed.

Qt (https://www.qt.io/) version 5.13.0 (newer versions might work, but are untested).

Wix Toolset version 3.11.2 or better.

obs-studio-dependencies.zip

Required Plugins

"obs-autosplitter"

https://github.com/CommanderPho/obs-autosplitter

Automatically splits the recording video stream into user-specified lengths by stopping and starting the recording. This serves to create output video files that are 4 hours long.


Development:

I've outlined the process below, but you can also reference https://obsproject.com/wiki/Install-Instructions#windows-build-directions.


After cloning the git repo, you'll need to use CMake to build the Visual Studio solution which can be opened in Visual Studio to start development.

Source Code: "C:/Users/halechr/repo/obs-studio"

Where to build the binaries: "C:/Users/halechr/repo/obs-studio/build64"

Click "Configure". Errors will be generated.

Set "QTDIR" to "C:/Qt/5.13.0/msvc2017_64"

Set "DepsPath" to wherever you extracted the above "obs-studio-dependences.zip" folder. For me it's "C:/Users/halechr/repo/obs-studio-dependencies/win64"

Click the "Configure" button again, and it shouldn't show an error this time and should end in "Configuring done". There is still a lot of red text and things that look like errors.

Then click "Generate". Should say "Generating done".

You're done in CMake, you can now close it.



Open the "build64/obs-studio.sln" file in Visual Studio. Make sure "Release" is selected up at the top (instead of "Debug") and the architecture is "x64". Build it by right clicking the "ALL_BUILD" item in the Solution Explorer and clicking "Build". The output is built in ".\build64\rundir\Release\bin\64bit\"


You'll useÂ Wix Toolset and the "PACKAGE" build target in Visual Studio to build the installer .msi.Â
